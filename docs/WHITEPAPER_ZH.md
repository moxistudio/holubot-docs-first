# 葫芦宝（HoluBot）白皮书（脱敏版）

更新日期：2026-02-28

## 1. 一句话定义

HoluBot 是 AI Agent 生态的治理与调度层（Control Plane）：把“调度”和“执行”解耦，在多通道、多后端场景下提供审批、审计、成本控制与可回滚能力。

## 2. 问题陈述

在 Agent 生态快速增长的背景下，团队普遍遇到以下问题：

- 执行权限过大，缺少关键动作前的人工确认
- 工具与记忆膨胀导致延迟和 token 成本失控
- 发生降级或故障时缺乏对用户的透明反馈
- 多后端并行时，缺少统一治理与审计层

## 3. HoluBot 的核心策略

- 轻前台：前台只负责理解、路由、治理，不承担重执行
- 重后端：复杂任务交给可插拔后端 Agent 执行
- 人在回路：高风险动作默认确认，信任级别可升可降
- 审计优先：关键动作可追踪、可复盘、可回滚

## 4. 架构原则

1. 前台永远不胖：能力边界稳定，不因后端扩展而膨胀
2. Skill 驱动：通过规范化流程降低对强模型的依赖
3. 信任由用户授予：系统不自行提升高风险权限
4. 异常透明：降级与失败必须向用户可见
